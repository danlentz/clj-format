# CLHS: Section 22.3 - Formatted Output ## Overview The `format` function generates nicely formatted text and messages. It can return a string or output to a specified destination. The control-string can be either a format string or a function (such as one produced by the `formatter` macro). ## Control String Structure A format control string comprises simple text and embedded directives. The `format` function writes simple text as-is, while directives specify additional output at corresponding positions. ### Directive Syntax A directive consists of: - A tilde (~) - Optional prefix parameters (separated by commas) - Optional colon and at-sign modifiers - A single character indicating directive type As the specification notes: "Prefix parameters are notated as signed (sign is optional) decimal numbers, or as a single-quote followed by a character." ### Parameter Options - **V or v**: Takes an argument from args as a parameter (should be an integer or character) - **#**: Represents the number of remaining arguments to process ### Example Directives - `~S` - S directive with no parameters or modifiers - `~3,-4:@s` - S directive with parameters 3 and -4, plus colon and at-sign flags - `~,+4S` - First parameter omitted (default), second parameter is 4 ## Output Destinations The destination parameter controls where output is sent: - **nil**: Returns a new string containing the output - **String with fill pointer**: Appends output to the string - **Stream**: Sends output to that stream - **t**: Sends output to standard output ## Subsections The specification includes detailed sections on: - FORMAT Basic Output - FORMAT Radix Control - FORMAT Floating-Point Printers - FORMAT Printer Operations - FORMAT Pretty Printer Operations - FORMAT Layout Control - FORMAT Control-Flow Operations - FORMAT Miscellaneous Operations - FORMAT Miscellaneous Pseudo-Operations - Additional Information about FORMAT Operations - Examples of FORMAT - Notes about FORMAT --- Source: http://www.lispworks.com/documentation/HyperSpec/Body/22_c.htm
